# Extracting Individual COBOL Programs (Official Method)

This project uses the **official GnuCOBOL method** to split the NIST COBOL 85 test suite (`newcob.val`) into separate `.CBL` files. No custom scripts are used for the split itself.

## How It Works (GnuCOBOL tests/cobol85)

1. **EXEC85** is a COBOL program that lives inside `newcob.val`. It is generated by the same `sed` rules used in GnuCOBOL’s `tests/cobol85/Makefile.am`: lines from `newcob.val` between `*HEADER,COBOL,EXEC85` and `*END` are taken and a few FILE-CONTROL lines are rewritten so that:
   - The population file is read from the path in the `NEWCOB_VAL` environment variable.
   - Output goes to `newcob.tmp` and `newcob.log`, and the control file is `EXEC85.conf`.

2. **EXEC85** is compiled with GnuCOBOL (`cobc -std=cobol85 -debug -x EXEC85.cob`).

3. For each **module** (NC, SM, IC, SQ, RL, ST, SG, OB, IF, RW, DB, IX, CM):
   - A directory `./<MODULE>/` is created.
   - `EXEC85.conf` is written with the first line `*SELECT-MODULE <MODULE>` and the rest taken from the official **EXEC85.conf.in** (from GnuCOBOL).
   - EXEC85 is run from that directory with `NEWCOB_VAL` set to the absolute path of `newcob.val`. It reads the population file and the config and writes all programs for that module into `newcob.tmp` in the module directory.

4. **expand.pl** (official script from GnuCOBOL `tests/cobol85/expand.pl`) reads `newcob.tmp` and splits it into:
   - One `.CBL` file per main program in the module directory.
   - Subprograms in `lib/*.CBL` when applicable.
   - Copybooks in `copy/` and `copyalt/` when applicable.

So the only “extraction” logic is: **EXEC85** (from NIST/GnuCOBOL) + **expand.pl** (from GnuCOBOL). The Makefile only drives these steps.

## Requirements

- **GnuCOBOL** (`cobc`) – to compile and run EXEC85.
- **Perl** – to run `expand.pl`.
- **newcob.val** – in the project root (from `newcob.val.tar.gz`).

## Commands

From the project root:

- **Extract all modules:**  
  `make all`  
  or  
  `make extract-all`

- **Extract one module (e.g. NC):**  
  `make extract-NC`

- **List extracted programs per module:**  
  `make list`

- **Clean (remove all module dirs and EXEC85):**  
  `make clean`

- **Regenerate EXEC85.cob from newcob.val:**  
  `make EXEC85.cob`  
  (then `make EXEC85` to recompile)

## Files Used (Official Sources)

| File            | Role |
|-----------------|------|
| `newcob.val`    | NIST CCVS85 population file (single archive of all programs). |
| `EXEC85.cob`    | Generated from `newcob.val` with the GnuCOBOL Makefile’s `sed` rules (see our Makefile). |
| `EXEC85.conf.in`| From GnuCOBOL `tests/cobol85/EXEC85.conf.in`. |
| `expand.pl`     | From GnuCOBOL `tests/cobol85/expand.pl`. |

Reference: [GnuCOBOL tests/cobol85](https://github.com/rupurt/GnuCOBOL/tree/master/tests/cobol85) and the same directory in the official GnuCOBOL repository.
